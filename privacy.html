Privacy Policy

Your data, your control

Last updated: November 27, 2025

Overview

XCred is a browser extension that displays publicly available account transparency information on X/Twitter. This policy explains what data we access and how we handle it.

Data Access

  • Account location ("Account based in")
  • Connection platform (iOS, Android, Web)
  • VPN/Proxy detection status
  • Verification status
  • Account creation date
  • Username change history

This is the same info visible on any account's "About" page on X.

  • Your X/Twitter login credentials

  • Your IP address or device info
  • Data from accounts you don't view

Data Storage

Local Cache (Your Browser)

Profile data is cached locally using IndexedDB to reduce API calls and improve performance. This data stays on your device and expires after 24 hours.

Shared Cache (Optional)

If "Remote Sync" is enabled in settings, anonymized profile data is shared with other XCred users via our database to reduce X API rate limits. Cached profiles expire after 7 days and are automatically deleted. You can disable this anytime.

Extension Client ID

A randomly generated identifier (e.g., "xcred_abc123") is created on first install and stored locally. This helps prevent spam and abuse of the shared cache. No personal information is tied to this ID.

P2P Cache (Optional)

If "P2P Sync" is enabled in settings, profile data is shared peer-to-peer with other XCred users via a decentralized relay network. This reduces load on our servers and X's API. P2P data is validated through peer consensus before being accepted. You can disable this anytime in extension settings.

Data Security

Local data is stored in your browser's IndexedDB, protected by your browser's security model. Shared cache data is stored in Supabase with multiple security layers:

  • Database Constraints: Validates all data format before accepting
  • Anomaly Detection: Automatically flags and removes suspicious data
  • Row-Level Security: Policies control who can read/write cached profiles
  • Automated Cleanup: Hourly jobs remove flagged and expired data

Data Retention

We automatically delete data according to these schedules:

  • Local Cache: Profiles expire after 24 hours; error entries after 30 minutes
  • Shared Cache: Profiles older than 7 days are automatically deleted
  • Anomaly Logs: Flagged profiles are removed within 24 hours of detection

You can manually clear all local data at any time via the extension settings.

Third-Party Services

Supabase (Remote Cache)

Used for the optional shared profile cache. Stores only public profile transparency data (location, platform, verification status). No personal user data is stored.

Gun.js P2P Network (Optional)

Used for optional peer-to-peer profile caching. Connects to a relay server to share cached profile data with other XCred users. Only public profile transparency data is shared. No personal user data is transmitted. Can be disabled via "P2P Sync" toggle in settings.

Data Sharing Practices
  • We do not sell any data
  • We do not share data with advertisers
  • Cached profile data may be shared with other XCred users (if Remote Sync is enabled) to improve performance
  • All shared data is already publicly accessible on X

We do not use any analytics, tracking, or advertising services.

Additional Information

Children's Privacy

XCred is not intended for use by children under 13. We do not knowingly collect data from children.
