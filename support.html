How It Works

XCred analyzes publicly available account information from X/Twitter's transparency data and displays a country flag with a color-coded border below each user's avatar. The flag shows the account's registered location, while the border color indicates a credibility tier based on multiple factors.

Understanding Credibility Tiers

Government Officially verified government accounts
Tier 1 (Blue) Strong authenticity signals
Tier 2 (Green) Good authenticity signals
Tier 3 (Yellow) Some inconsistencies detected
Tier 4 (Orange) Multiple concerns identified
Tier 5 (Red) Significant red flags present
Tier 6 (Gray) Insufficient data available

Scoring factors include: verification status, connection platform (iOS/Android/Web), location consistency, account age, VPN/proxy usage, and username change history.

Frequently Asked Questions

What do the colored borders mean?

The border color around each flag indicates the account's credibility tier. Silver borders indicate government accounts, blue indicates the highest credibility, and the scale goes through green, yellow, orange, red, and gray (insufficient data) as credibility decreases.

What factors determine the credibility score?
  • Verification status ‚Äî Government, Business, or Blue checkmark
  • Connection platform ‚Äî iOS, Android, or Web client
  • Location consistency ‚Äî Match between account and app store location
  • Account age ‚Äî Older accounts score higher
  • VPN/Proxy usage ‚Äî X flags accounts using VPNs
  • Username changes ‚Äî Frequent changes reduce score
Why doesn't an account show a flag?

Some accounts may not have location data available in X's public transparency information. XCred can only display data that X makes publicly available through their API.

Does XCred track me?

No. XCred does not track your activity, collect your personal information, or access your login credentials. All processing happens locally in your browser. See our Privacy Policy for complete details.

Will this slow down X/Twitter?

XCred is optimized for performance:

  • Remote Sync ‚Äî Shares cached data with other users to reduce API calls
  • Local caching ‚Äî Data is stored locally for faster subsequent loads
  • Rate limiting ‚Äî Built-in throttling prevents excessive API calls

For best performance, keep "Remote Sync" enabled in the extension settings.

Troubleshooting

Flags not appearing after installation
  1. Refresh the X/Twitter page (Ctrl+R or Cmd+R)
  2. Make sure you're logged into X/Twitter
  3. Check that the extension is enabled in chrome://extensions/
  4. Click the XCred icon to verify settings
  5. Try disabling and re-enabling the extension
Extension not updating data
  1. Go to chrome://extensions/
  2. Find XCred and click the refresh icon
  3. Reload any open X/Twitter tabs
Getting rate limited by X's API

If you're seeing rate limit errors:

  1. Open the XCred popup (click the extension icon)
  2. Make sure "Remote Cache Sync" is enabled
  3. This shares cached data with other users, significantly reducing individual API calls
  4. Wait a few minutes for rate limits to reset
